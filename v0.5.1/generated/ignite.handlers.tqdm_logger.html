ProgressBar

TQDM progress bar handler to log training progress and computed metrics.
class ignite.handlers.tqdm_logger.ProgressBar(persist=False, bar_format='{desc}[{n_fmt}/{total_fmt}] {percentage:3.0f}%|{bar}{postfix} [{elapsed}<{remaining}]', **tqdm_kwargs)[source]#

TQDM progress bar handler to log training progress and computed metrics.

Parameters

  • persist (bool) ‚Äì set to True to persist the progress bar after completion (default = False)

  • bar_format (Optional[str]) ‚Äì Specify a custom bar string formatting. May impact performance. [default: ‚Äò{desc}[{n_fmt}/{total_fmt}] {percentage:3.0f}%|{bar}{postfix} [{elapsed}<{remaining}]‚Äô]. Set to None to use tqdm default bar formatting: ‚Äò{l_bar}{bar}{r_bar}‚Äô, where l_bar=‚Äô{desc}: {percentage:3.0f}%|‚Äô and r_bar=‚Äô| {n_fmt}/{total_fmt} [{elapsed}<{remaining}, {rate_fmt}{postfix}]‚Äô. For more details on the formatting, see tqdm docs.

  • tqdm_kwargs (Any) ‚Äì kwargs passed to tqdm progress bar. By default, progress bar description displays ‚ÄúEpoch [5/10]‚Äù where 5 is the current epoch and 10 is the number of epochs; however, if max_epochs are set to 1, the progress bar instead displays ‚ÄúIteration: [5/10]‚Äù. If tqdm_kwargs defines desc, e.g. ‚ÄúPredictions‚Äù, than the description is ‚ÄúPredictions [5/10]‚Äù if number of epochs is more than one otherwise it is simply ‚ÄúPredictions‚Äù.

Examples

Simple progress bar

trainer = create_supervised_trainer(model, optimizer, loss)

pbar = ProgressBar()
pbar.attach(trainer)

# Progress bar will looks like
# Epoch [2/50]: [64/128]  50%|‚ñà‚ñà‚ñà‚ñà‚ñà      [06:17<12:34]

Log output to a file instead of stderr (tqdm‚Äôs default output)

trainer = create_supervised_trainer(model, optimizer, loss)

log_file = open("output.log", "w")
pbar = ProgressBar(file=log_file)
pbar.attach(trainer)

Attach metrics that already have been computed at ITERATION_COMPLETED (such as RunningAverage)

trainer = create_supervised_trainer(model, optimizer, loss)

RunningAverage(output_transform=lambda x: x).attach(trainer, 'loss')

pbar = ProgressBar()
pbar.attach(trainer, ['loss'])

# Progress bar will looks like
# Epoch [2/50]: [64/128]  50%|‚ñà‚ñà‚ñà‚ñà‚ñà      , loss=0.123 [06:17<12:34]

Directly attach the engine‚Äôs output

trainer = create_supervised_trainer(model, optimizer, loss)

pbar = ProgressBar()
pbar.attach(trainer, output_transform=lambda x: {'loss': x})

# Progress bar will looks like
# Epoch [2/50]: [64/128]  50%|‚ñà‚ñà‚ñà‚ñà‚ñà      , loss=0.123 [06:17<12:34]

Example where the State Attributes trainer.state.alpha and trainer.state.beta are also logged along with the NLL and Accuracy after each iteration:

pbar.attach(
    trainer,
    metric_names=["nll", "accuracy"],
    state_attributes=["alpha", "beta"],
)

Note

When attaching the progress bar to an engine, it is recommended that you replace every print operation in the engine‚Äôs handlers triggered every iteration with pbar.log_message to guarantee the correct format of the stdout.

Note

When using inside jupyter notebook, ProgressBar automatically uses tqdm_notebook. For correct rendering, please install ipywidgets. Due to tqdm notebook bugs, bar format may be needed to be set to an empty string value.

Changed in version 0.4.7: attach now accepts an optional list of state_attributes

attach(engine, metric_names=None, output_transform=None, event_name=Events.ITERATION_COMPLETED, closing_event_name=Events.EPOCH_COMPLETED, state_attributes=None)[source]#

Attaches the progress bar to an engine object.

Parameters

  • engine (Engine) ‚Äì engine object.

  • metric_names (Optional[Union[str, List[str]]]) ‚Äì list of metric names to plot or a string ‚Äúall‚Äù to plot all available metrics.

  • output_transform (Optional[Callable]) ‚Äì a function to select what you want to print from the engine‚Äôs output. This function may return either a dictionary with entries in the format of {name: value}, or a single scalar, which will be displayed with the default name output.

  • event_name (Union[Events, CallableEventWithFilter]) ‚Äì event‚Äôs name on which the progress bar advances. Valid events are from Events.

  • closing_event_name (Union[Events, CallableEventWithFilter]) ‚Äì event‚Äôs name on which the progress bar is closed. Valid events are from Events.

  • state_attributes (Optional[List[str]]) ‚Äì list of attributes of the trainer.state to plot.

Return type

None

Note

Accepted output value types are numbers, 0d and 1d torch tensors and strings.
